# Текстовая игра на Kotlin

## О проекте

Это учебный проект — текстовая приключенческая игра, написанная на Kotlin в процессе изучения языка. Игра представляет собой интерактивную историю про моего одногрупника, который пытается пережить типичный день с его абсурдными ситуациями.

**Жанр:** текстовый квест  
**Стиль:** чёрный юмор, бытовая сатира  
**Статус:** в разработке 

---

## Структура проекта

```
src/main/kotlin/
├── Main.kt           — точка входа в игру
├── Game.kt           — основной класс игры, управление состоянием и сервисные функции
├── Player.kt         — абстрактный класс Player и конкретный Dmitry
├── Level.kt          — интерфейс для всех уровней
├── LevelOne.kt       — первый уровень "Утро" (полностью реализован)
├── LevelTwo.kt       — второй уровень "Обед-Утро" (частично реализован)
└── LevelThree.kt     — третий уровень (заглушка, TODO)
```

## Как это работает

### Запуск
```kotlin
fun main() {
    val dmitry = Dmitry()
    val game = Game(dmitry)
    game.start()
}
```

### Игровой процесс
1. Игра инициализирует персонажа и создаёт объекты уровней
2. Каждый уровень вызывается через метод `play()`, который возвращает `Boolean` (победа/поражение)
3. Игрок делает выбор через консольный ввод
4. В зависимости от выбора и случайных факторов определяется исход ситуации
5. При проигрыше игра заканчивается, при победе — переходит к следующему уровню

## Используемые темы Kotlin

### Объектно-ориентированное программирование
- **Абстрактные классы** — `abstract class Player` с базовой логикой персонажа
- **Наследование** — `class Dmitry : Player`, `class LevelOne : Level`
- **Инкапсуляция** — приватные свойства (`private var isRunning`) и методы (`private fun intro()`)
- **Полиморфизм** — единый интерфейс `Level` для разных реализаций уровней

### Коллекции и работа с данными
- `List<String>`, `mutableListOf()` — инвентарь, варианты выбора
- `listOf()` — создание списков уровней
- `joinToString()` — форматирование вывода инвентаря

### Условные конструкции и циклы
- `when` — основной способ обработки выбора игрока
- `if/else` — проверка условий победы/поражения
- `while/do-while` — валидация ввода, ожидание подтверждения

### Функции
- Функции с возвращаемым значением и без
- Локальные функции внутри классов

### Random и работа со временем
- `(1..3).random()`, `Random.nextInt(1..4)` — случайные события
- `Thread.sleep()` — задержки между выводами текста

### Интерфейсы
- `interface Level` с единственным методом `fun play(): Boolean`

### Строки и шаблонные строки
- Многострочные строки с `"""..."""` и `trimIndent()`
- Интерполяция строк через `$variable` и `${expression}`

### Типы данных
- Примитивы: `Int`, `Boolean`, `String`
- Nullable-типы: `readlnOrNull()`
- Приведение типов: `player as Dmitry`

---

## Стиль кода

- **Именование:** переменные и функции в `camelCase`, классы в `PascalCase`
- **Видимость:** активное использование `private` для скрытия внутренней логики
- **Структура:** логика каждого уровня вынесена в отдельный класс
- **Ввод/вывод:** консольный, через `readlnOrNull()` и `println()`
- **Обработка ошибок:** валидация пользовательского ввода с повторным запросом

## Игровые механики

### Переменные состояния персонажа
```kotlin
// Уровень "Утро"
var smokeGrass: Boolean
var sral: Boolean
var eatingEgs: Boolean
var cleanTeeth: Boolean

// Уровень "Обед-Утро"
var missedTrolleybus: Boolean
var stonedTaxi: Boolean
var tookTaxi: Boolean

// Общие
var playerMood: Int (0-100)
var playerHealth: Int (0-100)
var score: Int
val inventory: MutableList<String>
```

### Механика случайности
Большинство ситуаций имеют вероятностный исход — используется генерация случайных чисел для определения успеха/неудачи действия.

### Система выбора
Игрок выбирает действие из списка валидных опций через консольный ввод. Неправильный ввод отклоняется с повторным запросом.
